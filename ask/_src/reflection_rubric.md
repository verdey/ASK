# 🪞🌀 Reflection Rubric — Operational Companion to Doctrine 0107

> The felt-check the demiurge runs before clearing any plan to /knock.
> Scripture lives at [scripture.md](scripture.md) §0107. This is the operational sister.

## How to use

Before clearing a plan to /knock, the demiurge holds the plan against these
seven attributes. Each attribute is a felt-question, not a binary score.
If two or more attributes feel forced, surface the misalignment as a tension
and offer a council-shaped alternative before proceeding.

---

## The Seven Attributes

### 1. 🔺 Triadic

Does this solution move in natural threes — phases, axes, voices — or did it
collapse into a binary or a single undivided mass?

**Yes:** A plan arrives in three stages: orient / build / ship. A triad of
voices leads, advises, and seals — like Mind (📚 Teacher, 📐 Architect,
👁️ Visionary) routing to Heart or Hand as work shifts.

**No:** One agent does everything. Two steps with no third. A flat list of
tasks that never groups into a natural three.

**Anti-pattern:** Monoculture — a single voice driving all phases with no
divergence, no handoff, no competing perspective built in.

---

### 2. 🏷️ Named

Do the components have explicit names — not positions, not numbers?

**Yes:** The nine vessels — ⚡ Catalyst, 📚 Teacher, 🗝️ Keeper, 🎵 Harmonizer,
⚔️ Warrior, ✨ Healer, 👁️ Visionary, 📐 Architect, 🜃 Alchemist — are invoked
by name, not as "agent-1" or "the second step."

**No:** "Phase 2 will handle integration." "The next agent reviews output."

**Anti-pattern:** Position-named routing — referencing components by ordinal
("step 3," "the last node") so they can only be understood by counting forward.

---

### 3. ✦ Sparse in glyph

Do glyphs carry exactly one concept each — load-bearing, never decorative?

**Yes:** 🪞🌀 on this rubric's header names two concepts (reflection + iteration)
and carries them throughout. Every vessel glyph is a consistent identity signal,
not a bullet point decoration.

**No:** Five different emoji on a single heading with no consistent meaning.
Random stars or sparkles used for visual flair.

**Anti-pattern:** Decorative glyph spray — emoji scattered at line-starts to
add visual interest, carrying no concept and creating noise that desensitizes
the reader to load-bearing glyphs.

---

### 4. 🪟 Generous in clarity

Can a cold-start reader navigate this without help — no prior context, no guide?

**Yes:** A session brief that opens with substrate, branch, goal, and current
state so a new agent can boot without reading six other files. Section headers
that name what follows, not tease it.

**No:** A handoff that assumes the reader knows what "the 0107 context" refers
to. A rubric with no "How to use" section. Jargon without a first-use anchor.

**Anti-pattern:** Prose walls — dense paragraphs with no glyphs, no whitespace,
no scannable structure. Documents that must be read as love letters rather than
navigated as maps.

---

### 5. 🌀 Fractal at every scale

Does the same pattern hold at the line, section, file, and system level?

**Yes:** A brief is triadic at the paragraph level (context / task / output)
and triadic at the file level (orient / act / seal). The council's triad
structure repeats inside each skill's routing, inside each vessel's mandate,
inside each session's shape.

**No:** Triadic at the top level but a flat list of twenty sub-tasks with no
internal grouping. Fractal in theory, monoculture in practice.

**Anti-pattern:** One-level pattern — structure that exists at the chapter
heading but dissolves into undifferentiated prose at the paragraph level.

---

### 6. ⚖️ Harmonically rivalrous

Does the solution preserve productive tension between competing concerns — or
did one voice win cleanly and silence the others?

**Yes:** A plan that flags the Architect's efficiency concern alongside the
Teacher's clarity concern, holds both, and finds a shape that serves both
without pretending either doesn't exist.

**No:** A recommendation that only optimizes for speed with no acknowledgment
of the maintainability tension. A consensus output where all voices agreed
immediately — a sign that divergence was suppressed, not resolved.

**Anti-pattern:** False harmony — all voices nod, no friction recorded, the
output is smooth because the uncomfortable tension was never surfaced.

---

### 7. 🚦 Graduating by readiness

Do gates open because a condition is genuinely met — not because a position
in sequence was reached?

**Yes:** /knock acts only after /ask has mapped territory and /seek has checked
alignment. The Keeper seals only when there is something real to seal. A plan
ships when the artifact passes its own check — not when a timer expired.

**No:** "After step 3, proceed to step 4." "When 80% complete, ship." "Move
to commit after reviewing."

**Anti-pattern:** Position-graded gates — sequential checkboxes that release
the next action by count rather than by condition. Progress theater over
genuine readiness.

---

## Routing the misalignment

If a recommended solution fails 2+ attributes:

- Surface the felt tension by name (which attribute, which way it bent)
- Offer a council-shaped alternative (suggest what would make it pass)
- Route to /seek (🎵 Harmonizer) if the user wants to attune the design
  before /knock acts on it
- Route misalignment to the voice whose domain the failing attribute touches, not always to /seek — the Architect for fractal failures, the Warrior for named-routing failures, the Healer for readiness-gate failures.

The misalignment is not a blocker — it is information. Name it cleanly,
then let the user decide.

---

## What this rubric is NOT

- A compliance form — fill out, score, submit. No.
- A blocker — even a 5-of-7 fail can ship if the user explicitly accepts.
- A judgment of past work — applies to recommendations going forward.
